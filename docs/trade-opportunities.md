# Trade Opportunities

Trade Opportunities is a **local feature on a single page, driven by two on-demand jobs**: **Discover Trade Opportunities (DTO)** finds brand-new investment ideas through deep web research and hard data analysis, organizing them across a fixed risk-by-horizon matrix grounded in the current Market Signal house view; **Audit Trade Opportunities (ATO)** re-evaluates the existing matrix opportunities the user selects. Like Portfolio Analysis both run entirely on local models (see [local-models.md](local-models.md)) and are **prescriptive and firm**; unlike it, neither is **tied to current holdings** — the purpose is to discover and maintain new opportunities. Both **require a connected Schwab account**, whose option chains supply the per-candidate options-activity signal (see [schwab-integration.md](schwab-integration.md)).

One invariant frames the whole lifecycle and is worth stating up front: **only a deep re-evaluation can archive an opportunity.** The cheap re-derivation — engine-only, run on every live opportunity DTO doesn't deep-research, and offered on demand in ATO — refreshes the quant read and *flags* a name for attention, but it can **never** archive one; deep research is the sole destructive path. This is what lets the cheap pass run liberally over the whole matrix while the consequential action stays gated (see [§Archived opportunities](#archived-opportunities)).

## What the job hunts

The goal is profit: ideally **finding a name before it moves**, but also **catching a move early enough that even a "late" entry compounds** (a memory-cycle or AI-infrastructure name six months into its run still returned multiples). These are two distinct detection problems, and the job runs **both modes**:

- **Early detection** — a *leading operating metric inflecting before the income statement and before the multiple re-rates*, while a live bear narrative still suppresses the price (e.g. a high-margin segment re-accelerating at a depressed multiple; a free-cash-flow turn while the market prices bankruptcy; a first surprise profit into a crowded short).
- **Continuation** — *demand-visibility signals* that license buying a move already underway: an estimate-revision and beat-and-raise streak, a guidance step-change far above consensus, rising backlog / book-to-bill, corroborating customer (hyperscaler) capex, or a commodity contract-price turn with supply still constrained.

The discipline that separates a real opportunity from a chased story is **the leading-metric anchor**: a candidate's narrative must be tied to a *countable, dated, third-party-verifiable* number that is moving — not the story alone. A narrative with no inflecting operating metric behind it is a story stock, and is rejected. This is the same data-honesty stance as the report's evidence floor and the suite's compute-don't-guess rule ([local-models.md §Context-memory discipline](local-models.md#context-memory-discipline)), applied to discovery.

## The two jobs

Both jobs are **manual**, run in the run tracker, and share the **single global run slot** with the report and Portfolio Analysis — only one runs at a time across the app (see [local-models.md §Failure posture](local-models.md#failure-posture), [run-tracking.md](run-tracking.md)). They share one page (the 3×3 matrix and the archive) and split the lifecycle by purpose:

- **Discover Trade Opportunities (DTO)** — the generative job, triggered by a single **Discover** button, with per-cell progress in the run tracker. It runs the full discovery funnel ([§The pipeline](#the-pipeline)), **deep-researches the discovered candidates** under one compute budget — a **small reserved rotation slice first** (configurable, default ~20% of the budget, never rounding below one slot) spent on live opportunities in **maintenance-priority order** — warning-bearing names first (a tripped falsifier or continuation break shouldn't queue behind an uneventful stale name), then catalyst proximity, then threshold-near, then stalest by `last_deep_researched_at`, backstopped by a **max-age service level** so no live name's research can age past a configured bound whether or not it re-surfaced (the matrix's self-refresh) — then **new names first** in the remainder, with any **leftover budget** spent on existing opportunities that re-surfaced through discovery (oldest-deep-researched first) — and then **cheap re-derives every other live opportunity** so the whole matrix is refreshed each run. Deep research is the only path that archives; the cheap sweep only raises warnings. A candidate that resolves to a name **already live** in the matrix is reconciled against the existing record rather than re-discovered blind (the re-surface either earns a leftover-budget deep pass or falls to the cheap sweep); a candidate matching a **departed (archived)** ticker is a fresh re-entry ([§Archived opportunities](#archived-opportunities)).
- **Audit Trade Opportunities (ATO)** — the maintenance job, **user-directed**. The user **selects one or more existing opportunities** on the matrix (per-card selection plus select-all / deselect-all), which enables the **Audit** button; clicking it opens a two-option fork — **Quick Audit** (the cheap re-derivation) or **Deep Audit** (the full deep re-evaluation). Quick Audit refreshes the quant read and may raise a warning but never archives; Deep Audit is the manual route to a fresh deep pass and can archive. A large Deep-Audit selection prompts a confirmation first (it runs the per-candidate loop on every selected name — potentially long), in the same spirit as the SearXNG pre-run consent ([web-research.md](web-research.md)).

The **cheap re-derivation** and the **deep re-evaluation** are the same two engine/model paths in both jobs — DTO invokes them automatically (cheap on the matrix tail, deep on discovered candidates), ATO on the user's selection. Within a single DTO run a ticker is deep-researched **at most once**: a name deep-researched as a discovered candidate is never also cheap-swept, and a re-surfaced name that won a deep pass is not re-derived again.

## The opportunity space

Output is organized as a **3×3 matrix**: three **risk tiers** (high / medium / low) × three **horizons** (short / mid / long term) = nine cells, each holding **however many opportunities clear the gates, ranked by conviction** — or none, when nothing qualifies. The gates set a cell's count, not a quota: a cell is as full or as empty as the evidence earns. The user sees high-, medium-, and low-risk sections, each containing short-, mid-, and long-term ideas. **Risk-tier assignment is deterministic** — derived by rule from measurable inputs (profitability, market cap, liquidity, volatility, leverage, drawdown, and event exposure), not a label the model picks — so the same asset lands in the same tier run to run. The fixed matrix is what makes the output comparable across runs and forces breadth — the job must consider every risk/horizon combination rather than clustering on whatever is topical.

## Archetype — the lens that decides which signals matter

Risk×horizon *places* an opportunity; **archetype decides how to find and judge it.** The same signal means opposite things in different kinds of business — a low trailing P/E is a *buy* tell for a steady compounder and a *top-of-cycle sell* tell for a commodity cyclical — so the job classifies each candidate into one of five archetypes, and that classification selects the **signal weighting and the valuation lens** for the rest of the pipeline. Archetype is a first-class dimension (deterministic features, model-confirmed), feeding the risk-tier rule rather than replacing it:

| Archetype | The leading tell to detect | Valuation lens | "Late but still works" continuation tell |
|---|---|---|---|
| **Secular compounder** | a high-margin recurring/platform engine accelerating *inside* a mispriced business | PEG; estimate-revisions-vs-multiple | margin/segment reveal + base breakout |
| **AI / secular-cyclical infra** | segment-revenue YoY acceleration; estimate-revision velocity | forward P/E *against* the revision rate | beat-and-raise streak, guidance step-change, customer-capex corroboration, backlog/book-to-bill |
| **Commodity cyclical** | supply discipline + spot/contract-price turn at washed-out sentiment, *while still loss-making* | **P/B, P/NAV, mid-cycle EPS — trailing P/E suppressed** | contract-price up-quarter with supply still cut; a new demand kicker |
| **Category disruptor** | a leading operating metric (reservations, subscriber net-adds, trial efficacy, scanner share) + a hard demand>supply proxy + a strategic-incumbent stake | track the leading metric, not EPS | net-add / guidance beats — *and the inverse as the exit* |
| **Quality compounder** | operating income decoupling upward from revenue; durability-under-stress (e.g. renewal rate holding through price hikes) | valuation is a **risk gate, not an entry** | margin stair-steps; recurring profit pool growing toward a third of profit |

Archetype also informs the deterministic risk-tier (quality compounders skew lower-risk; commodity cyclicals and disruptors skew higher) and the horizon read. For a **carried-forward name the archetype is sticky**: the confirm call receives the prior label and must affirm it unless the deterministic classification features have decisively moved — an overturn must cite a changed feature the app can verify in the input delta, else the prior label holds ([trade-opportunities-workflow.md §Step 5a](trade-opportunities-workflow.md#step-5a-archetype-classification)) — so the valuation lens and composite weights can't churn between passes on a borderline name.

## The research method

The job's edge is not a screen — it is a **way of researching**. Most screens find what is already cheap or already moving; this job tries to find what the world is *about to* reward, by first building a view of **how the world is evolving** — where industries are heading, which hard problems are being solved, where durable value is being created — and only then combining that worldview with the hard numbers and the live news to choose. The discipline is to think like several different researchers at once and require that they agree: a quant, an economist, a fundamental business analyst, and a market historian each interrogate a candidate through their own lens, and an opportunity earns a place only when the **conjunction** holds — never a single signal (base rates are brutal: most stocks underperform Treasuries over their life, and the traits of the winners recur in the losers).

### The worldview comes first

Before any name is judged, the job grounds itself in a **forward worldview**, in two layers:

- the **macro / regime backbone** is inherited from the Market Signal house view — the report already commits to a `market_cycle` (late-cycle / recessionary / recovery) and a `risk_posture` (risk-on / off / mixed) ([storage.md](storage.md)), a compact growth × inflation regime read that says which sectors carry a tailwind in this phase of the cycle. The job does **not** recompute it; it reuses it (subject to the one-week freshness gate — [trade-opportunities-workflow.md §Step 2](trade-opportunities-workflow.md#step-2-load-shared-context)).
- the **forward thematic map** is the job's own work: the active secular themes (AI compute, electrification, reshoring, GLP-1 / obesity, the energy transition, aging demographics, …), each placed on its **adoption S-curve** — durable and compounding when penetration is still low and the **unit-cost curve is falling** (a falling cost expands the addressable market irreversibly — the Wright's-Law engine that turns a niche into a mass market), but merely speculative when it is a story with no cost or penetration anchor. Each theme is then traced down its **value chain** to the specific beneficiaries — **economically, not just by exposure**: at each node the tracing asks *who actually captures the margin* (where the chain's economic-profit pool sits, not merely who touches the theme), *who holds the bargaining power* (supplier- vs buyer-concentration — a node that can dictate price versus one squeezed between a strong supplier and a strong customer), *where capacity is constrained* (the bottleneck that confers durable pricing power — a sold-out fab, a permitted mine, a single-source component), and *which enabler therefore has real pricing power* rather than only revenue exposure — deliberately *past* the crowded pure-plays to the **picks-and-shovels enablers** (suppliers, infrastructure, cost-reducers) sitting at those margin-capturing, constrained nodes, whose fortunes rise with the adoption curve but whose names the consensus hasn't yet bid up. Exposure without margin capture — a commoditized supplier into a powerful buyer — is a thematic trap, not an opportunity. This is where breadth across market caps pays off: the enabler is frequently a mid- or small-cap.

The worldview is what lets the job answer "what *will* be valuable," not only "what *is* cheap." It is a bias, never a verdict: a theme still has to produce a name with a moving number behind it.

### The lenses

Each candidate is interrogated through five lenses. The first two are **engine-computed** (numbers the engine derives, never invented by the model — [local-models.md §Context-memory discipline](local-models.md#context-memory-discipline)); the last three are built by the **research loop** over the open web, grounded by those numbers.

- **Quant lens (engine).** An integrated **composite** — each factor normalized against sector-adjusted absolute bands and the name's own history ([trade-opportunities-workflow.md §Step 5c](trade-opportunities-workflow.md#step-5c-deterministic-analysis-archetype-weighted-engine)) — over the factors that survive out-of-sample — **value** (earnings / FCF / EBIT-to-EV), **momentum** (12-month-minus-1 price *and* the name's own time-series trend), **quality** anchored on **gross profitability** (revenue − COGS over assets, the least-polluted quality metric) plus profitability / growth / safety, **low beta**, and **conservative investment** (low asset growth) — scored as *one* composite rather than separate sleeves, so a value tilt isn't secretly short momentum. On top sit the **event / flow signals**: **estimate-revision breadth** ((up − down) / total), **earnings surprise (SUE)** and its post-announcement drift, **insider cluster buys** (open-market, multiple insiders — strongest in small caps), **institutional accumulation**, and **short interest** (a **bearish factor by default** — heavily-shorted names underperform on average, since short sellers are informed — flipping to a *squeeze* long only as a narrow **conditional** setup: an inflecting leading metric **and** a near-term catalyst **and** evidence the bear thesis is breaking, never high short interest alone). Breadth across **all market caps** is the point (a modest signal applied to many names beats a strong signal on a few) — but with the universe's `*-bulk` pre-scoring off-plan and the screener carrying no fundamental field to rank a universe on, the composite is computed **per candidate on the narrowed set** ([trade-opportunities-workflow.md §Step 5c](trade-opportunities-workflow.md#step-5c-deterministic-analysis-archetype-weighted-engine)), so breadth is protected **structurally** — by the cap-band-stratified discovery selection that feeds the composite ([trade-opportunities-workflow.md §Step 4](trade-opportunities-workflow.md#step-4-candidate-consolidation)) — rather than by scoring the whole cross-section. The composite keeps faith with that breadth: **size used only within quality** (small-*and*-high-quality, never raw small-cap) and a **tradability flag** (illiquidity / days-to-cover) attached rather than excluding small names outright.
- **Value-creation lens (engine).** Whether the business actually *creates* value rather than just growing: **ROIC above its cost of capital** by a real spread (growth funded below the cost of capital destroys value however fast revenue grows), with **R&D capitalized** so research-heavy leaders aren't mis-scored as unprofitable; the **reinvestment runway** (g ≈ ROIC × reinvestment rate); **owner earnings**; and the **moat** that protects the returns — classified by source (intangibles, switching costs, network effects, cost advantage, efficient scale) and weighted by how often each actually sustains a durable advantage. High returns with no moat mean-revert; the moat is the reason the future cash flows survive.
- **Macro / thematic-fit lens (research).** How the candidate sits in the worldview above: which theme it rides, where that theme is on its S-curve, whether it is a crowded pure-play or an **un-priced enabler at a margin-capturing, capacity-constrained node** (exposure alone is not the tell — pricing power is), and whether the **TAM** holds up when sized **bottom-up** (units × price) and not only top-down (the gap between the two methods is itself a narrative tell). The leading indicators an economist watches that front-run the income statement — PMIs, capex commentary, book-to-bill, freight, the semiconductor cycle, commodity supply / demand — are gathered here, since no structured feed carries most of them.
- **Investor-judgment lens (research).** The qualitative read the great stock-pickers built by **scuttlebutt**: management quality and capital-allocation behavior (insider buying, buybacks, guidance *delivered* vs *promised*, candor in bad quarters), the durability of the growth, and the **pre-consensus** tells that mark an early edge — thin analyst coverage, low institutional ownership, a thesis that diverges from what the price already reflects (a *variant perception*).
- **Pattern / case-study lens (research).** The candidate is checked against the **library of how the historical winners actually started** — and against the look-alikes that failed. The library is a **curated, versioned data asset shipped with the app** — structured episodes (name, period, archetype, the tell's actual dated metric series, how it resolved), not prompt prose — supplied to the lens by **retrieval, never model recall**: a local reasoner recalling historical specifics unaided will fabricate them confidently. The same asset doubles as the **gate regression harness** ([§Starting parameters](#starting-parameters-calibratable)), and to keep that second use honest it ships in **three partitions**: a *grounding* set (the memorable canon this lens retrieves), a *development* set (the only episodes gate constants may be shaped against), and a **locked holdout** — ordinary failures, near misses, and randomly sampled base-rate names, not memorable cases — never tuned against. The recurring early tells: a **high-margin segment compounding inside a lower-margin whole** before the market re-rates it (AWS inside Amazon, Services inside Apple, Azure inside Microsoft); an **oligopoly with supply discipline turning** (memory ASPs up, inventories down, backlog rising — Micron); a **usage / engagement metric inflecting ahead of revenue** (Netflix streaming hours); a **platform shift the incumbent is mispricing**; **new management + a credible guidance step-change** (Nadella's Azure); **forward customer commitments** (long-term supply agreements); and **replicable single-unit economics with a long whitespace runway** (Walmart, Costco — both small-caps when the tell was visible). The inverse is the red-flag set the selection gate enforces (see [§The pipeline](#the-pipeline)): a multiple outrunning estimates (Cisco 2000), peak-cycle margins extrapolated (memory tops, Peloton over-build), demand pull-forward mistaken for trend (Peloton / Zoom 2021), a narrative with no leading metric (dot-coms), a deteriorating leading metric behind still-strong share (Nokia / BlackBerry), or earnings-quality games (GE Capital).

### The two non-negotiables

Across every lens, two rules are load-bearing because they are the cleanest dividers between the winners and the failures:

1. **A present, inflecting leading operating metric is mandatory.** Every historical winner had a *countable, dated, third-party-verifiable* number already moving — segment revenue accelerating, ASPs turning, renewal rates holding, net-adds rising — before the price did; every failure either lacked one or had one quietly rolling over. A narrative with no inflecting metric behind it is a story stock and is rejected ([§What the job hunts](#what-the-job-hunts)). This is the anti-narrative gate, and it applies equally to early-detection and continuation ideas. *Inflecting* is **shaped by the metric's family, not one curve** — acceleration for growth metrics, a turn-that-holds for cyclical prices, stability-under-stress or a widening decouple for durability metrics — with the archetype mapping to the family its tell lives in, deltas taken seasonally comparable (YoY for quarterly series), steady post-inflection strength sufficing for a continuation idea, and a noise floor scaled to the series' own variability; the operative rule is drafted in [§Starting parameters](#starting-parameters-calibratable).
2. **Valuation-vs-forward-estimate trajectory is the primary red-flag.** A move that is mostly **multiple expansion** — price and multiple outrunning flat or declining estimates — is the signature of nearly every late-stage blow-up, so the **narrative-vs-reality ratio** (revision pace vs multiple change) caps or excludes such a name rather than promoting it, even when the company is excellent. For **thinly-covered small- and mid-caps** — frequently the very enabler names breadth prizes — analyst estimates are absent or stale, so the ratio falls back to an **operating-reality-vs-price** read: the hard operating momentum the company itself *reports* (segment revenue, backlog / bookings, gross profit, unit economics, cohort retention — the archetype's leading metric, the same series the evidence floor accepts in place of GAAP statements for emerging-economics archetypes — [§Evidence floor](#evidence-floor)) measured against the move in price and multiple, in place of the street's revisions. The question is unchanged — is the move underwritten by improving economics, or is it multiple expansion *ahead* of them? — only the numerator shifts from analyst revisions to the firm's own operating series when coverage is too thin to read.

### Reconciling the lenses (the contradiction check)

Requiring the lenses to *agree* — the conjunction rule this section opens with — only bites if their disagreement is actually surfaced, and the per-topic research design deliberately keeps the lenses **isolated in separate clean contexts** until they are consolidated, precisely so the local model is never handed a sprawling prompt. The consolidation is therefore also the **adjudication**: when the per-topic findings are distilled into one object, the same pass explicitly reconciles them — *which lenses disagree* (a strong macro / thematic story over a failing value-creation or management read; a quant composite that likes a name whose unit economics don't support it), *what would falsify the thesis* (the specific, monitorable **key falsifiers**), and *whether any source is stale or conflicted*. This is **folded into the existing distillation and scoring stages — no extra model call**: distillation emits a typed cross-lens contradiction (with a severity) and the falsifiers alongside the synthesis; scoring must then *resolve or discount* an unreconciled contradiction rather than average it away; and the deterministic gate **caps — never promotes — a high-severity contradiction** (a strong theme over a weak business). The point is to stop a single loud lens, usually the narrative, from carrying a name the other four don't support — and the falsifiers it produces are exactly what the continuity exit signals and the outcome-learning pass ([§Outcome learning](#outcome-learning-calibration)) later test against, typed by re-check class so the cheap sweep can evaluate them deterministically ([§The opportunity](#the-opportunity)).

### Balancing the revolutionary against the proven

The job hunts both the *revolutionary* company and the one with simply *excellent numbers* — and these are not opposites but the **same equation calibrated to different points on the compounding curve**: every durable winner is roughly *(high or credibly-emerging returns on capital) × (a long reinvestment runway) × (a moat) × (a price that still leaves asymmetry)*. The difference is only **where the return is proven**. The *proven-economics* track (Buffett / Greenblatt / Marks) requires it already visible in trailing financials and demands a margin of safety; the *emerging-economics* track (Fisher, Lynch's fast growers, disruptive-innovation and crossover-growth investing) underwrites it in a forward TAM × penetration × margin model and substitutes a **return hurdle** (the idea must be able to roughly double over its horizon) for trailing cheapness. The **archetype** ([§Archetype](#archetype--the-lens-that-decides-which-signals-matter)) routes a candidate onto the right track and sets the weights — a commodity cyclical judged on mid-cycle earnings and supply discipline, a secular compounder on its margin / segment engine and runway — but **both tracks pass through the same moat / management / price-asymmetry gate**. A company with strong numbers need not be revolutionary to be a real opportunity; a revolutionary one must still clear the price gate and show the inflecting metric. Holding that balance — not collapsing into either pure momentum-chasing or pure deep-value — is the job's stance.

The two tracks also differ in **how research moves conviction** — the job's expression of the suite's **four-part model** (the engine's backward anchor — quant composite + value-creation read · the **forward outlook** — scenario targets + thematic-fit · **conviction**, the bidirectional synthesis · the matrix placement that follows). The discipline gates below always *cap or lower* conviction; but on the **emerging-economics track** — where the forward inflection legitimately precedes the income statement — research may also **raise** it, bounded, when it validates a *countable, dated, third-party leading indicator the engine's structured feeds did not already score* (a `research_metric`-class signal — DRAM/NAND ASPs, channel checks, transcript backlog). The **proven-economics track keeps its margin-of-safety discipline**: a strong forward read informs the outlook and the targets, but trailing cheapness still governs the call. The **governing invariant** is identical to Portfolio's — a conviction *raise* must trace to an **engine-unscored** leading indicator, **never** to price action or narrative, and never to a metric the engine already scored at Step 5c (the no-double-count guard that keeps this distinct from the cap-only since-flagged read — [§Outcome learning](#outcome-learning-calibration)). *Engine-unscored* means **absent from the Step-5c composite / leading-metric series** (a `research_metric` with no structured feed) — the test is whether the engine already counted it, **not** whether it is the mandatory leading-metric anchor: a structured anchor already in 5c cannot also raise conviction, while a research-only signal it could not score can. The raise is **bounded (≤ one level)** and carried as a typed **`validated_leading_indicator`** the app-layer gate validates ([trade-opportunities-workflow.md §Step 5h](trade-opportunities-workflow.md#step-5h-deterministic-risk-tier-gate-validation--checkpoint)), so it is never prompt-only.

### The event-impact / value-chain repricing lens

A discrete **technology, product, or standard announcement** — a new chip, a process node, a regulatory standard, a reference design — reprices a whole value chain at once, and the market's first reaction is routinely wrong in *both* directions: it oversells names whose real exposure is narrow and overbuys names whose benefit is thin. The job treats such an event as a first-class, **two-sided** research trigger rather than letting it enter only as ambient news. The lens is deliberately named for **repricing, not "disruption"** — the central question the research must *derive*, never assume, is whether the new technology **substitutes for, complements, or merely shifts the mix of** each affected business; disruption is only one of those answers (the same tech can complement an incumbent, shift its mix, change capex timing, create a supplier bottleneck, or simply be misread).

When the materiality gate (below) fires, the route maps the chain three ways:

- **beneficiaries** — who captures the new margin: the bottleneck, the hardware / systems, the suppliers and service layers the shift creates (the same picks-and-shovels-enabler tracing as the worldview lane, aimed at a specific event — e.g. the firms that build the hardware for the large closed-loop cooling systems a new chip's thermal design now requires);
- **feared losers** — the names that sold off: *what revenue / profit pool is actually exposed*, whether the technology substitutes / complements / touches only a narrow segment, and over what deployment timeline — so a panic sell-off with limited real exposure surfaces as an **early-detection** candidate (a live bear narrative suppressing the price — [§What the job hunts](#what-the-job-hunts)), while a genuinely impaired one is correctly left alone;
- **latent names** — companies that did **not** move but should be affected if the technical read is right (a cheap output of the value-chain trace: the chain is already enumerated, so the lens just asks which nodes the market hasn't repriced yet).

The same read governs the **upside**: a name that *rose* on the announcement but whose benefit is overstated is the mirror case and is **capped through this lens**, not waved through — the narrative-vs-reality ratio is a revisions-vs-multiple discipline and does not by itself read the technology and size the real exposure ([§The two non-negotiables](#the-two-non-negotiables)).

**The materiality gate** keeps this from chasing every product headline: the trigger is a material technology / product / standard announcement **plus** at least one corroborating condition — meaningful group repricing, credible primary-source documentation, a customer-adoption signal, or clear value-chain exposure. Absent the gate the route stays dormant, so a normal run behaves exactly as before.

**The "actual technology read" is a structured forcing-function, not narrative.** Each affected name carries a typed **`technology_read`** — `{ technical claim, deployment timeline, substitute / complement / mix-shift, affected workload or use case, exposed revenue / profit estimate, adoption constraints, switching costs, the margin-capturing node, source confidence, leading metric to monitor }` — so the panic-vs-real (or euphoria-vs-real) verdict is *sized*, anchored, and auditable rather than a vibe (e.g. "the new cooling addresses only the liquid-to-chip segment; the incumbent earns ~80% from air-side units that are unaffected"). The read rides the existing **hypothesis card** as an optional sub-object, so it flows through scoring, the adversarial passes, and promotion unchanged. The feared-loser side adds **one symmetric adversarial pass** to the lane's existing set — *is the impairment real or panic, and what is the actually-exposed pool?* — the mirror of the beneficiary-side "why is the obvious beneficiary wrong?".

This is **one signal among many**: every name the lens surfaces still clears the leading-metric anchor, the narrative-vs-reality and forensic gates, and the per-cell validation like any other candidate. The lens decides *what to interrogate when an event repriced a chain* — not *what to buy*.

### Why the funnel and the discipline gates

Because the richest lenses are research (expensive, and the model's context is finite) and the base rates are unforgiving, the job is a **funnel**: cheap quant and thematic screens narrow the universe, and only the survivors get the full five-lens treatment. Three discipline gates run before any candidate is promoted ([§The pipeline](#the-pipeline)): the **forensic gate** (accruals quality, Altman Z, Piotroski F, and manipulation-style flags — a value trap or an earnings-manager is excluded, not graded around); a **reflexivity check** (a thesis that depends on a self-reinforcing price → sentiment → fundamentals loop continuing is flagged, because it reverses violently when the loop breaks); and the **base-rate conjunction** rule (score the agreement of independent lenses, never one signal, starting from the outside view of how often this *kind* of setup has actually worked). Factor *timing* is deliberately avoided — the evidence that it works is weak — so the lens weights are archetype-conditioned but otherwise stable run to run, not regime-switched.

## The pipeline

The job runs as a **funnel**: broad, cheap discovery narrows to a small candidate set, then each candidate gets expensive per-name validation — the condense-as-you-go discipline the rest of the suite uses, and the shape the data budget requires (the richest signals are rate-limited, so they are spent only on the narrowed set).

1. **Hypothesis & candidate discovery (research generates candidates).** Three converging feeders, the first two research-active — this is where ideas are *found*, not merely vetted:
   - **Model-led hypothesis research** — the 122B reasoner, primed with the house view, **plans research routes** (policy / supply-chain / technical-bottleneck / procurement-capex / customer-capex / industry-history / failure-analogue / event-impact value-chain-repricing) and runs the **keyless SearXNG web tool** plus FMP Articles + structured-news seeds and the macro-release calendar (**no GDELT, no Tavily** — see [trade-opportunities-workflow.md §Step 3b](trade-opportunities-workflow.md#step-3b-model-led-hypothesis-research)) to form **investable hypotheses** — *world-change → mechanism → who captures the margin → leading metric → public-company expressions → bear case → falsifiers* — and only then **reasons its way to specific names**, tracing each hypothesis down its value chain to the margin-capturing operators and surfacing under-covered / early-inflection names no structured feed carries (each grounded in source URLs and verified before it earns budget). Hypotheses are **scored before any ticker** (so names come from strong hypotheses, not loud narratives) and vetted by **adversarial passes** (*why already priced? · why is the obvious beneficiary wrong? · who captures the margin instead?*); one route each run is a mandatory **graph-blind outside-view** so the discovery memory aids without anchoring. This is the **early-detection** lane and the job's edge: names that aren't yet loud in the data.
   - **Bottom-up structured** — cheap, broad, deterministic: the **company screener** slices the eligible universe by market-cap band / sector / liquidity (coarse fields only — no valuation or growth filter, and the `*-bulk` universe-scoring endpoints are off-plan, so it **stratifies rather than pre-scores** — [data-sources.md §FMP — current paid-plan tier audit](data-sources.md#fmp--current-paid-plan-tier-audit)); **commodity-price turns** (FRED / Stooq) for the cyclical sleeve; and **event / positioning feeds** — insider-buy clusters (FMP), short-interest extremes (FINRA), M&A / 8-K / movers / earnings-calendar / IPO catalysts (the activist-stake and congressional feeds are **symbol-keyed** on the current plan — no market-wide read — so they enter per-candidate at validation, never as discovery feeders — [data-sources.md §Trade Opportunities — endpoint surface](data-sources.md#trade-opportunities--endpoint-surface)), and a **post-earnings surprise screen** — the paid earnings calendar read *backward* over the trailing weeks (it carries consensus + actuals), surfacing **standardized** positive surprises (scaled by the name's own surprise history, SUE-style, not raw % — and prioritized where the **revenue surprise agrees with the EPS surprise**, where post-earnings drift is markedly stronger) and building beat-and-raise-streak candidates: the one structured feeder aimed squarely at **continuation** mode, whose tells (revision streaks, surprise momentum) are otherwise computed only per-candidate, after something else has already surfaced the name. These surface names **already visible in the data**; the multi-factor composite and forensic gate that rank them are computed **per candidate** after narrowing ([trade-opportunities-workflow.md §Step 5c](trade-opportunities-workflow.md#step-5c-deterministic-analysis-archetype-weighted-engine)), not across the universe.
   - **Carried-forward watchlist** — the persisted **opportunity graph** ([§Discovery memory](#discovery-memory-the-opportunity-graph)) re-surfaces every worthy-but-unpicked name from prior runs whose **leading metric is now confirming**, so a deferred idea that quietly starts compounding is caught rather than lost to chance.

   The combined set is deduped, sanity-filtered for tradability, tagged with the signals that surfaced it, and **narrowed to the per-candidate research budget** (a generous, **user-configurable** compute ceiling on how many names get the expensive validation per run — *not* a quality cap; nothing is validated yet at this point, so an un-picked name is **not rejected, only deferred** — and a genuinely worthy deferral is **remembered**: written to the persisted opportunity graph as a watchlist node and re-checked every later run, its leading metric monitored, so silent compounding can't slip away (the discovery-memory carry-forward — upstream of, and distinct from, the persisted *validated* matrix's carry-forward) — [configuration.md §Local Analysis Suite Configuration](configuration.md#local-analysis-suite-configuration)) **under deliberate diversity guardrails** — quotas and floors by market-cap band (a mid / small-cap floor and a mega-cap ceiling), feeder, archetype, and sector / theme — so the funnel stays broad rather than collapsing onto mega-cap momentum, the most-covered AI names, or one crowded macro theme (the breadth across market caps that is the job's edge; the consolidation spec is [trade-opportunities-workflow.md §Step 4](trade-opportunities-workflow.md#step-4-candidate-consolidation)). (This is the load-bearing fix to the old ordering: "names surfaced by research" is a *discovery* feeder that runs **here**, before candidate selection — research is no longer only a downstream validation step.)
2. **Archetype classification.** Each candidate is tagged with its archetype (deterministic features + model confirmation), selecting the signal weights and valuation lens for the steps below.
3. **Deterministic analysis (the engine computes every number).** Per candidate, the Rust financial-analysis engine (shared with Portfolio Analysis — see [portfolio-analysis.md](portfolio-analysis.md)) computes the archetype-appropriate quantitative picture: the leading-metric series (revision velocity, segment acceleration, commodity-price turn, margin decoupling), the earnings surprise / SUE, positioning (insider net, short interest, COT for cyclicals, the Schwab options signal), scenario price targets, a **price-action confirmation read** (relative strength vs the market / sector and proximity to a multi-year base breakout, from equity price history), **and three derived reads the selection stage leans on** — the **narrative-vs-reality ratio** (estimate-revision pace vs multiple change: *justified-expensive* when estimates outrun the multiple, *hype* when the multiple outruns flat/declining estimates), the **forensic flags** (margin compression while revenue accelerates; net-income-vs-operating-cash-flow divergence; receivable/inventory build outpacing sales; restatement / auditor-change history), and the **implied-expectations read** — the growth / revision trajectory the current price *already assumes*, solved by inverting the engine's scenario math at the live price under the archetype's valuation lens and expressed as a **range under stated assumptions** (a price is consistent with many growth × margin × discount combinations, so a single solved number would be false precision), so "what must be true to justify today's price" is a computed, bounded read the priced-in / crowding judgment anchors on rather than a vibe. The model interprets these; it never invents them.
4. **Deep web research** — the 122B reasoner (thinking mode) plus the web tool, running the bounded per-topic research loop per candidate ([web-research.md](web-research.md)), fail-soft. This is where the three research lenses of [§The research method](#the-research-method) are worked, each as its own clean-context topic so the local model is never overwhelmed: the **macro / thematic-fit** read (which theme, where on the S-curve, pure-play vs enabler, bottom-up TAM, the economist's leading indicators); the **investor-judgment** read (the **driving narrative and market sentiment** — how much of the price reflects emotion about *what might come* versus present fundamentals — plus management / capital-allocation quality and the pre-consensus tells); the **pattern / case-study** read against the early-winner library; the external corroboration the structured feeds can't give (customer / hyperscaler capex commentary, supply discipline, transcript backlog / TAM targets / inflection language, and **DRAM/NAND ASP direction** — the one cyclical price with no free structured feed); and, **mandatory, the contemporaneous bear case** — the job must state why a name might fail, because the winning traits also appeared in the famous failures (Cisco, Intel, GE rode the same surface signals down). Every claim is grounded by the engine's quant and value-creation numbers so research fills the gaps the numbers can't, rather than substituting a story for them.
5. **Distillation** — the reasoner in non-thinking mode consolidates the loop's findings into candidate summaries (the same resident 122B by default; the fast 35B tier is a benchmark-gated option — see [local-models.md §The model roster and per-task routing](local-models.md#the-model-roster-and-per-task-routing)).
6. **Selection, scoring & gating.** The 122B reasoner interprets the computed analysis and distilled research and selects per cell, under explicit discipline: **score the conjunction, never a single signal** (base rates are brutal — most stocks underperform Treasuries over their life, and the winner traits recur in losers); **require the leading-metric anchor plus external validation**; **apply the narrative-vs-reality ratio**; treat **price action (relative strength / base breakout) as a confirmation overlay** that adjusts conviction but never substitutes for the leading-metric anchor; and **run the risk / forensic gate** — a candidate tripping the forensic flags, or whose move is mostly multiple-expansion, is capped or excluded rather than promoted. Each survivor is assigned its deterministic risk tier (archetype-informed) and horizon → matrix cell, and **every survivor that clears the gate is listed there, ranked by conviction** (deduped to genuinely distinct ideas — near-identical peers collapse to the best-expressed one) — the gate sets the count, not a quota, and the **app layer validates the listing is complete** (every gated survivor present, or explicitly collapsed into a named peer with a recorded reason), so the per-cell step is a ranking + dedup pass, not a model gatekeeper that could quietly drop a name. **A cell may return no opportunity** when nothing qualifies — empty cells are honest, not failures, and the matrix never pads itself to fill them; a rich cell that genuinely earned many anchored ideas is just as honest, never trimmed to a fixed number.
7. **Continuity & re-evaluation (DTO).** Prior opportunities are carried forward with an updated status; additions and `invalidated` departures must be justified by what changed (see [thesis-continuity.md](thesis-continuity.md)). DTO splits the carried matrix by cost: an opportunity that **re-surfaced through discovery and won leftover deep-research budget** is **deep-re-evaluated in the Step-5 validation loop as a carried-forward candidate** (the only path that can flip it to `invalidated` → archive), while at the continuity step **every other live opportunity gets the engine-only cheap re-derivation** — it re-pulls price + estimates, re-derives the scenario targets (decaying any stale research prop), re-runs the entry asymmetry gate, and refreshes the since-flagged read, but it **never archives**: an exhausted-upside or continuation-failure signal it detects (revisions rolling over, a beat-streak breaking, shipments diverging below sell-through) raises a **warning** on the card (*consider a Deep Audit*), not an exit. There is **no "target met" exit**; a name leaves the matrix for the **archive** only when a deep re-evaluation — here, or in ATO — judges it `invalidated` ([§Archived opportunities](#archived-opportunities)). Each prior opportunity also receives its **deterministic outcome label** in this step, and the **continuous since-flagged read** is refreshed across the whole matrix ([§Outcome learning](#outcome-learning-calibration)).

## Discovery memory (the opportunity graph)

Discovery is **stateful**: each run persists an **opportunity graph** — the structured trace of the model's worldview reasoning *and* the memory of worthy candidates that didn't make this run's cut. Its shape is `theme / hypothesis → economic mechanism → value-chain node → leading metric → companies → evidence → key falsifiers`, so the macro / political / structural reasoning behind every name is **auditable and reusable across runs** instead of evaporating into prose. A hypothesis surfaced by the **event-impact repricing lens** additionally carries its typed **`technology_read`** and the side it expresses (beneficiary / feared-loser / latent — [§The event-impact / value-chain repricing lens](#the-event-impact--value-chain-repricing-lens)), so the sized panic-vs-real read persists with the node and can be re-checked next run like any other watchlist evidence.

Company nodes carry a **status**:
- **picked** — became a validated opportunity in this run's matrix (the matrix is the authoritative carry-forward for these; the graph preserves the hypothesis lineage behind the pick);
- **watchlist** — cleared the **app-enforced discovery-worthiness bar** (a named / countable / dated leading metric + a stated economic mechanism + ≥1 falsifier + a hypothesis score above threshold — [trade-opportunities-workflow.md §Step 3b](trade-opportunities-workflow.md#step-3b-model-led-hypothesis-research)) but didn't earn a validation slot (budget / diversity), or was researched but didn't clear a gate while its thesis stays live. Carried forward with its hypothesis lineage, leading metric (tagged by **re-check class** — `structured` / `filing` / `research`), key falsifiers, and a *why-deferred* reason, and **re-checked at its metric's cadence every later run** ([trade-opportunities-workflow.md §Step 3c](trade-opportunities-workflow.md#step-3c-carried-forward-watchlist-re-check)): if the leading metric is **confirming**, the node is promoted as a priority discovery feeder;
- **retired** — falsifiers tripped, the leading metric went dead / stale, the thesis was invalidated, or the **carry horizon** elapsed (a configurable cap on how long a hypothesis is monitored without confirming — counted in the leading metric's **own reporting periods** (drafted: ~4), never in runs: run cadence is user-chosen, so a run-counted horizon would give a quarterly metric days at daily cadence and years at monthly). Retired nodes leave active monitoring but stay in history for outcome-learning.

This directly fixes the failure mode of stateless re-discovery — a deferred name that quietly compounds but never re-trips a feeder, and so is silently lost (there is no promise it re-surfaces). It is **bounded by construction**: a retention cap on the active watchlist plus the self-pruning of falsified / stale / horizon-elapsed nodes ([configuration.md §Local Analysis Suite Configuration](configuration.md#local-analysis-suite-configuration)). Statefulness's opposite risk — **anchoring** to stale theses — is countered by the mandatory **graph-blind outside-view route** in discovery ([trade-opportunities-workflow.md §Step 3b](trade-opportunities-workflow.md#step-3b-model-led-hypothesis-research)) and the carry horizon, so the memory aids without caging. The graph is **distinct from the validated opportunity matrix**: the graph is *upstream* discovery memory (hypotheses + worthy-but-unpicked candidates), the matrix is the *downstream* validated output carried forward for the Step-7 continuity check. Both persist in this job's own storage partition ([storage.md §Local Analysis Suite Storage](storage.md#local-analysis-suite-storage)).

## Evidence floor

Discovery decides whether a candidate is *worth* the expensive per-name work; the evidence floor decides whether a validated candidate has earned a place in the matrix at all — the per-candidate analog of the report's Step-3 coverage floor and Portfolio's holding floor ([portfolio-analysis.md §Evidence floor](portfolio-analysis.md#evidence-floor)). Below the floor a candidate returns **`insufficient-evidence`** — an explicit abstention held out of the matrix ([trade-opportunities-workflow.md §Step 5h](trade-opportunities-workflow.md#step-5h-deterministic-risk-tier-gate-validation--checkpoint)) — rather than a low-conviction guess. The floor splits inputs by criticality, and — unlike Portfolio's single equity floor — it is **archetype-aware**, because an early-stage disruptor legitimately carries thinner financials than a quality compounder:

- **Floor-bearing for every candidate:** a current **quote / price** and **price history** (the engine's momentum / price-action reads); a **validated leading operating metric** — the inflecting, countable, dated, third-party-verifiable anchor the whole thesis hangs on ([§What the job hunts](#what-the-job-hunts)); and **source freshness** — the leading metric and the bear-case evidence must be current, not stale. A candidate with no inflecting leading metric is rejected as a story stock however complete its other data is: the leading-metric gate *is* part of the floor.
- **Statement floor, archetype-substituted:** the **financial statements** (FMP / SEC) are floor-bearing for a *proven-economics* archetype (quality / secular compounder, commodity cyclical) — absent, stale, or carrying conflicting identity ⇒ abstain. For an *emerging-economics* archetype (early disruptor, pre-profit ai-infra) where GAAP statements are thin by nature, a defined **substitute** stands in: hard **operating / unit economics** (segment revenue, backlog / bookings, net-adds, cohort retention, gross profit) sufficient to underwrite the leading metric, plus the mandatory bear case. The substitution is explicit and logged; it relaxes the *form* of the supporting financials, never the leading-metric requirement.
- **Enriching (never forces abstention):** estimates / revisions, positioning, peers, the options signal, analyst opinion, and the qualitative narrative read — their absence **lowers conviction** and is recorded as a degraded-input flag, but never on its own floors the candidate.

"Firm" means committing when the evidence supports it and abstaining when it doesn't — an empty cell is honest, and `insufficient-evidence` is a real verdict, not a failure ([§The pipeline](#the-pipeline)).

## The opportunity

Each opportunity is a structured, schema-validated record:

- **asset / ticker**
- **archetype** — secular-compounder / ai-infra / commodity-cyclical / disruptor / quality-compounder (the lens that judged it)
- **detection mode** — early or continuation
- **directional thesis** — firm and specific
- **leading operating metric** — the countable, dated anchor and its trend (the thing that must be moving)
- **catalyst** — why now
- **horizon** — short / mid / long (matches its cell — set from `expected_thesis_realization`; the separate `business_runway` read feeds durability / conviction, never the cell — [§Starting parameters](#starting-parameters-calibratable))
- **expected thesis realization** (`expected_thesis_realization`) — when the thesis *pays*: a dated payoff-bearing catalyst, the market's recognition of a re-rate, or the multi-year compounding itself — **rule-derived and recorded at Step 5h** (never model-picked, like the tier and horizon it drives)
- **business runway** (`business_runway`) — the durability read: how many years of high-return reinvestment remain, **engine-derived at Steps 5c / 5f** from the typed runway proxies (penetration + falling unit cost, contracted backlog coverage, or a multi-year `research_forward_assumption`); feeds durability and conviction, never the cell
- **risk tier** — high / medium / low, assigned deterministically by rule (matches its cell)
- **conviction** level
- **narrative-vs-reality read** — whether the move is fundamentally underwritten or multiple-expansion, from the estimate-revisions-vs-multiple ratio
- **forward outlook / price prediction** *(user-facing)* — the engine's base-case scenario **price target** and its bear / bull range over the engine's **fixed twelve-month target window** (a rolling ~12-month window from the run date — the same basis the shared engine computes for Portfolio, so targets are comparable across cells; **distinct from the matrix horizon**, which reflects catalyst timing, not the target window; as-built the field is still named `end_of_year` — the settled rolling-window rename, [portfolio-analysis.md §Starting parameters](portfolio-analysis.md#starting-parameters-calibratable)), methodology exposed, incorporating any research-sourced **`research_forward_assumption`** applied at [trade-opportunities-workflow.md §Step 5f](trade-opportunities-workflow.md#step-5f-deterministic-target-refinement). This is the *forward* read, surfaced **distinct from the backward engine anchor** (the quant composite + value-creation read) so a name that screens poorly on trailing numbers but carries a strong forward setup is shown as such, not buried ([§Balancing the revolutionary against the proven](#balancing-the-revolutionary-against-the-proven))
- **bear case** — the contemporaneous reason it might fail (mandatory)
- **key falsifiers** — the specific, monitorable conditions that would invalidate the thesis (from the cross-lens contradiction check — [§Reconciling the lenses](#reconciling-the-lenses-the-contradiction-check)), **each typed by the watchlist's re-check classes** ([trade-opportunities-workflow.md §Step 3c](trade-opportunities-workflow.md#step-3c-carried-forward-watchlist-re-check)): a **`structured`** falsifier is a machine-evaluable condition over an engine series (metric + comparator + threshold — *segment growth < 20% YoY* — plus **persistence semantics**: a materiality margin and a required consecutive-observation count — drafted 1 for filing-cadence series, where a filing print is the period's settled observation so the first qualifying breach **confirms immediately** (the materiality margin is its noise guard), and 2 for high-frequency ones, where a single noisy print logs a quiet first-breach note and only a **confirmed breach** trips the amber warning — with the breach streak **observation-identity-keyed** (per condition: last distinct observation id, last value and evaluation time, streak, first-breach and confirmation times, and — once a deep pass clears the warning a confirmed breach raised — the **acknowledging observation id**, so the warning re-raises only on a breach confirmed against a *later* observation, never straight back from the one the deep pass already examined; the streak advances only on a new print or filing, never on re-evaluating the same one, so repeated cheap passes against one quote can't confirm a breach — and keyed **across rewrites** by a stable, app-assigned **`condition_id`** under a structural-identity rule: a deep-pass rewrite that leaves the machine-evaluable core (metric, comparator, threshold, margin, count) unchanged carries the id and its state through any re-wording, while an edit to that core **supersedes** — the old condition closes with its state into the audit record and the successor starts a **fresh streak** with a `supersedes` link — decided by the app at the Step-5h validation, never asserted by the model, so a threshold edit can't inherit a streak accrued against a different threshold and a wording-only edit can't erase one)) that the app validates at Step 5h and the **cheap re-derivation then evaluates every run**; a **`filing`** falsifier re-checks when a fresh filing lands; a **`research`** falsifier (a qualitative / competitive break) is testable only by a deep pass — so the continuity tripwires and the outcome-learning pass test the thesis's *own* stated break-conditions deterministically, not only the generic signals
- **hypothesis lineage** — the discovery hypothesis this name expresses (world-change → mechanism → margin-capturing value-chain node → leading metric), linking the opportunity to its node in the **discovery memory** ([§Discovery memory](#discovery-memory-the-opportunity-graph)) so the worldview reasoning behind the pick is auditable and the name's pre-validation history (first-surfaced run, prior watchlist time) is recoverable, including any **seed lineage** — the structured-feed headlines (FMP news / articles, the macro-release calendar) that surfaced or oriented the hypothesis where it was feed-seeded, recorded as **leads, never scored as evidence** ([web-research.md §The research loop and context management](web-research.md#the-research-loop-and-context-management))
- **technology read** *(present only when the name was surfaced by the event-impact repricing lens — [§The event-impact / value-chain repricing lens](#the-event-impact--value-chain-repricing-lens))* — the typed `technology_read` carried from its hypothesis card (substitute / complement / mix-shift, the exposed revenue / profit pool, deployment timeline, switching costs, the margin-capturing node), so a panic-vs-real or euphoria-vs-real call is sized and auditable
- **entry consideration**
- **risk / forensic flags** — any tripped quality gate
- **status** — `new`, `still-valid`, or `invalidated`, for carry-forward across runs; an `invalidated` opportunity leaves the matrix for the **archive** ([§Archived opportunities](#archived-opportunities)), and **only a deep re-evaluation can set `invalidated`** (the cheap re-derivation never does). There is **no "target met" exit** — an opportunity persists while it still clears re-evaluation, however far its price has run
- **attention warning** *(engine-set, cleared by a deep pass)* — a non-destructive flag the cheap re-derivation raises when a live opportunity warrants a fresh deep look, surfaced on the card as **Consider Deep Audit**. It fires on any of three deterministic, deliberately **high-bar** triggers: a **tripwire** (a stored `structured`- or `filing`-class **key falsifier confirming a breach** per its persistence semantics — the thesis's own stated break-condition, the sharpest trigger of the set; a lone noisy print only logs a first-breach note — a forensic flag newly tripping, a since-flagged gain diverging hard from leading-metric continuation, the narrative-vs-reality read crossing into `hype`, the margin of safety compressed near the upside-exhaustion line, or a drawdown breach), an **upside-exhaustion** read (the re-derived base case — structured-only once any research prop has decayed — no longer clears the entry gate), or a **re-surfacing** (the name re-appeared in DTO discovery but didn't earn a leftover-budget deep pass). Decay alone does **not** raise it — research staleness is surfaced separately by the quiet **Research stale** badge (see `last_deep_researched_at` below), so the amber warning stays reserved for an actual problem and can't blanket the matrix on a timer. The warning never changes the verdict or archives the name — it only prompts a Deep Audit, and a deep pass (DTO or ATO) clears it
- **became-an-opportunity date** (`became_opportunity_at`) — the run it first entered the matrix (this **is** the *first-surfaced run* the outcome-learning calibration anchors on — [§Outcome learning](#outcome-learning-calibration) — the same date, named for the card; distinct from any prior watchlist time); the anchor for its since-flagged return and shown on both the live card and (after departure) the archive. A name re-discovered *out of the archive* starts fresh, so this resets; a name that merely re-surfaces while still live keeps it
- **last deep-research date** (`last_deep_researched_at`) — when its qualitative deep research (Step 5) was last refreshed. The cheap re-derivation runs on the matrix tail each DTO run, but a deep pass happens only on a rotation-slice pick or a leftover-budget re-surface (DTO) or a user-chosen **Deep Audit** (ATO), so this marks how fresh the qualitative read is (the quant read is always current). It drives three things: an opportunity deep-researched **earlier today** (the current calendar day in the user's local timezone) shows a green **Deep-researched today** confirmation; one whose last deep pass is older than the **~4-week** research-freshness window shows a quiet, informational **Research stale** badge (the visual counterpart of the decayed `research_forward_assumption` — distinct from the amber attention warning, which it never raises on its own); and the date's age sets the **continuity weight** the next deep pass applies to the prior record (see the re-evaluation paths below)
- **since-flagged performance** *(engine-attached, present once the idea has a prior run — i.e. carried-forward)* — the running return since it became an opportunity (`became_opportunity_at`; absolute and vs sector / market), the maximum drawdown over that window, and the leading-metric-continuation state, reconstructed from price history each run, with the matured-window 1/3/6/12-month labels attaching here as they elapse ([§Outcome learning](#outcome-learning-calibration))

The fixed archetype / risk / horizon / status vocabularies keep the matrix stable and the list evolving rather than churning — an idea persists with an updated status instead of silently reappearing or vanishing.

**What each re-evaluation path refreshes.** The record is touched by two paths of different reach and different power — the dividing line is that **only the deep path can archive** ([trade-opportunities-workflow.md §Step 7](trade-opportunities-workflow.md#step-7-continuity-check--carry-forward), [§Step 5](trade-opportunities-workflow.md#step-5-per-candidate-validation-loop)):

- The **cheap re-derivation** (engine-only — run on the matrix tail every DTO run, and on demand as ATO's **Quick Audit**) refreshes the **engine-computed fields**: the since-flagged performance, the leading-metric-continuation state, the narrative-vs-reality ratio, the forensic computations, the deterministic risk tier, and the re-derived forward target — and **evaluates the stored key falsifiers by class**: `structured`-class every run, `filing`-class when a fresh filing has landed since the last check (Step 3c's cadence discipline applied to the matrix); a **confirmed breach** (per the falsifier's persistence semantics) raises the attention warning — a lone noisy print only logs a first-breach note — never an archive, and the falsifier *text* stays frozen model-authored — only its evaluation is engine-run. It **never re-rates a survivor and never archives** — an exhausted-upside, continuation-failure, or tripwire signal it detects raises the non-destructive **attention warning** above instead of an exit. The **model-authored fields freeze between deep passes**: the directional thesis, **conviction**, bear case, key falsifiers, catalyst, archetype, the typed `technology_read`, and entry consideration all hold their last deep-pass values.
- The **deep re-evaluation** (the full Step-5 loop — run on a DTO rotation-slice pick (the maintenance-priority queue), a re-surfaced name that won leftover DTO budget, or a user-chosen ATO **Deep Audit**) re-interprets everything against fresh research, rewriting **both** kinds, producing a new `research_forward_assumption` (which restarts the decay window — [§Starting parameters](#starting-parameters-calibratable)), and is the **sole path that can judge a name `invalidated` → archive**. It is **time-aware about the prior record**: the dossier carries a derived **`continuity_weight`** from the age of `last_deep_researched_at` (a recent prior pass is framed as *continued research* to anchor on; an old one as *a prior view to test skeptically*), banded to the same ~4-week window the `research_forward_assumption` decays over ([§Starting parameters](#starting-parameters-calibratable)). The weighting only frames how hard the model leans on the prior record — every engine number is recomputed fresh regardless, and the anti-anchoring guards (the cap-only since-flagged read, the graph-blind discovery route) are untouched.

The **forward outlook / price prediction is the hybrid** that sits on neither side cleanly: its **quant skeleton** (the scenario / DCF math off estimates, the run-level risk-free discount rate, and fundamentals) is re-derived on **every cheap re-derivation** (the DTO matrix-tail sweep or an ATO Quick Audit) as well as inside a deep pass, while the **research layer** baked into it — the `research_forward_assumption` — is **frozen at the last deep pass and decays to structured-only** past its freshness window. So the displayed target moves whenever the engine re-derives it, but any *research-driven* lift inside it is carried from the last deep research, **never newly generated by the cheap check** — only a deep re-evaluation injects fresh research into the prediction. `last_deep_researched_at` is the card's marker for how old that research layer is (the quant read is always current).

## Signal inputs

## Continuity and isolation

The job retains its most recent N runs, feeds the prior run into the next, and embeds results into the **Trade Opportunities memory partition only** — isolated from the report's and Portfolio Analysis's memory (see [local-models.md §Run history and continuity](local-models.md#run-history-and-continuity)). Output is firm and does not churn between runs absent hard data.

## Outcome learning (calibration)

A continuity status (`invalidated` → archived) records *that* an idea stopped clearing re-evaluation; it does not record *how it actually paid off* or *why it failed* in a form the job can learn from. So each **DTO** run computes a **deterministic outcome label** for every prior opportunity old enough to have a forward window — and, in shadow, the same price read for every name the funnel affirmatively turned away (below) — turning the job's own track record into structured feedback rather than leaving its misses unexamined (an ATO audit refreshes the labels for the names it touches; the durable-learning write that feeds calibration is a DTO side-effect, so it stays idempotent rather than firing on every page render). The labels are **engine-computed, not model-judged** (the suite's compute-don't-guess stance applied to self-assessment), so calibration reads numbers, never vibes:

- **forward return at 1 / 3 / 6 / 12 months**, both absolute and **relative to its sector and the broad market**, computed over the window since the opportunity was first surfaced (its `became_opportunity_at` run — first matrix entry, not watchlist discovery) with the engine's existing price infrastructure (Stooq history plus a sector / market benchmark series);
- **maximum drawdown** over the window — the path, not only the endpoint (an idea that doubled after a 40% drawdown is a different lesson than a straight line);
- **did the leading metric continue?** — the engine re-pulls the anchor series (segment-revenue acceleration, revision velocity, the commodity turn) and records whether the inflection that justified the idea persisted, stalled, or reversed;
- **why it failed**, for the losers — assigned by a **deterministic decision tree** over the input deltas, never a free narrative: `leading-metric-rollover` (the anchor reversed), `multiple-unwind` (price fell on de-rating while estimates held — a valuation miss, not a business one), `forensic-materialization` (a flagged quality risk came due), `thesis-played-out` (the move completed as expected — note this is an **internal-calibration outcome label for a winner**, distinct from any matrix status: hitting a target is not an exit, so there is no `played-out` *status*), or `market-beta` (the name fell with its group, the thesis intact). The accompanying explanation is **templated from the decision-tree branch**, so even the prose stays deterministic.

Those four are the **matured-window** labels — fixed, comparable checkpoints the calibration loop needs, each populated only once its window has actually elapsed. Alongside them the engine keeps a **continuous since-flagged read** in two parts. Its **price-derived parts** — the running **absolute return, return vs sector and vs market, and running maximum drawdown** — are measured from the **first-surfaced price** (the first **Stooq** daily close **after** `became_opportunity_at` — the measurement contract's next-close evaluation anchor, below — the run the idea first entered the matrix, **not** any prior watchlist-discovery date — [§The opportunity](#the-opportunity)) to the **latest Stooq daily close**, reconstructed purely from **Stooq daily bars**; they are refreshed on each DTO run *and* **re-derived at render as a keyless zero-job freshness floor** — a local recompute over a per-ticker Stooq cache (refreshed at most once per day, [§Storage and display](#storage-and-display)), never a fetch on every open — so the displayed performance stays current even between runs. The **leading-metric-continuation state** is the read's other part and is **not** in that floor: it needs a structured re-pull, so it refreshes only on a cheap re-derivation or deep pass (a job), never at render. Unlike the horizon labels it is live from the **first run after the idea was added** (a brand-new pick carries no since-flagged read on its debut run — there is no elapsed window yet) — so an idea carries a scorecard from that run onward rather than waiting a month for the 1-month checkpoint — and it advances one step per run; because each run reconstructs the full daily series from the first-surfaced date, the curve carries every day's point regardless of run cadence (only its right edge moves forward per run). It is the same stateless reconstruction the horizon labels use (no stored snapshots, only the carry-forward identity and price history), so it is **one engine primitive with three readers**: the **discrete horizons feed calibration**, the **continuous read feeds the matrix display** (the since-flagged curve and its current numbers, rendered inline beside each carried-forward idea — [§Storage and display](#storage-and-display)), and — for a carried-forward candidate re-validated this run — it feeds **scoring as reflexivity-disciplined context** ([trade-opportunities-workflow.md §Step 5g](trade-opportunities-workflow.md#step-5g-scoring--gating)): a since-flagged gain unmatched by leading-metric progress reads as building multiple-unwind risk and **caps** conviction, a since-flagged drawdown with the metric intact reads as improved asymmetry — never a raw momentum boost. A gain *matched* by metric progress is held **neutral, not boosted** (the metric is already scored directly at Step 5c, so crediting the price gain too would double-count it), so the read is **cap-only by design** — it can hold or lower conviction, never raise it.

The opportunity's stored **key falsifiers** (from the cross-lens contradiction check — [§Reconciling the lenses](#reconciling-the-lenses-the-contradiction-check)) are the checklist this pass tests against, closing the loop from *what we said would break it* to *what actually did*. Outcome labels persist with the run's audit record and embed into the **Trade Opportunities memory partition** as durable learnings ([§Continuity and isolation](#continuity-and-isolation)); because the entry reference is reconstructed from price history at the first-surfaced date, no extra per-opportunity state is needed beyond the carry-forward identity.

**The scorecard extends to what the job did *not* pick — the misses are the larger, cheaper, and more informative half of calibration.** Labels over picks alone converge slowly (a handful of names per run, split across five archetypes and four maturity windows) and measure only half of selection skill — whether the gates *admit* winners, never whether they *reject* them; for a discovery job the costlier error is the **false negative**, the winner it saw and declined. So the same stateless price primitive also runs over a persisted **shadow ledger** of every name the funnel affirmatively judged and turned away: candidates the Step-5 loop validated but Step 5h held out (`insufficient-evidence`, a forensic hard-exclude, a `hype` exclusion — each with its recorded failing gate), Step-6 **dedup-collapsed peers**, and the opportunity graph's **retired and still-unpromoted watchlist nodes** ([§Discovery memory](#discovery-memory-the-opportunity-graph)). The entry unit is a **decision episode — one per ticker per turn-away, typed by decision class** — because these are *different decisions* and pooling them would confound the read: a **gate reject** is a real negative judgment, an **`insufficient-evidence` abstention** may only reflect missing data, a **budget / quota deferral** (a watchlist node) was never judged at all, a **dedup substitute** was accepted economically and merely collapsed for redundancy, and a **watchlist retirement** (`retired-hypothesis` — [storage.md §Local Analysis Suite Storage](storage.md#local-analysis-suite-storage)) is a distinct fifth decision — stopping the monitoring of a hypothesis, judged on its own anchors below. Each episode anchors on the run that turned it away — a **deferral** on its first-surfaced date (what did the budget / quota cost?), while a **retirement carries both that date and its retirement date** (was retiring right? — a different question, so both anchors persist) — and carries its surfacing tags, feeder / route lineage, archetype, and — for a Step-5h reject — the **full gate vector with each gate's distance from its threshold**, not just the first failing gate (gates apply in sequence, so first-failure attribution would be an artifact of evaluation order; the full vector makes *which gate actually caused a miss* answerable, including the counterfactual "had only gate X moved, would it have entered?"); the engine then computes the same **matured-window forward returns and drawdown, price-derived only** (absolute and vs sector / market, the archive's exact Stooq reconstruction — never a leading-metric re-pull, so a shadow entry costs no per-symbol budget and no model call). Two derived reads are the point, **computed per decision class, never pooled**: the headline **picked-vs-rejected spread** compares picks against the **gate rejects** (at each window, did the funnel's admissions outperform its true refusals?) — on **unique-issuer counts**, so a repeatedly re-rejected ticker can't inflate the sample, sliceable by feeder, route, archetype, and gate, so *where the edge lives* is answered empirically rather than assumed — while the other classes carry their own reads (deferrals price the budget / quota cost, abstentions price the evidence floor, dedup substitutes should track roughly with the pick that absorbed them, retirements test whether stopping the watch was right); and a **false-negative flag** on any turned-away name whose market-relative return clears the drafted bar ([§Starting parameters](#starting-parameters-calibratable)), **discounted by the tradability flag** — an untradeably illiquid "miss" is not a real miss. The ledger is **bounded by construction** — its own retention cap ([configuration.md §Local Analysis Suite Configuration](configuration.md#local-analysis-suite-configuration)), each entry **frozen into a compact matured archive** once its 12-month labels are recorded (matured episode-level rows persist under their own cap rather than being discarded — re-computing a spread or testing a future gate proposal against realized history needs the row-level data, which aggregates and embeddings can't recreate) — and **calibration-only under the same passive, anti-reflexive stance as the archive** ([§Archived opportunities](#archived-opportunities)): a shadow entry's performance never re-promotes, re-surfaces, or displays it as a live idea — a flagged false negative tunes the *gates*, and the name itself re-enters only through independent fresh discovery.

**Outcome measurement holds a point-in-time execution contract** — labels are learning data, so how a return is measured is part of the spec, not an implementation detail. The decision timestamp is the run; the entry reference is the **next session's Stooq close — a consistent evaluation anchor, deliberately not called an executable price** (the true first executable is the next open, but daily-bar opens are the least reliable field in the feed, and one consistent, conservative anchor matters more than intraday fidelity — never the same-day bar the decision could not have traded). Returns are split-adjusted, and **every cross-entry comparison runs on one common basis — price-only, the basis derivable for every entry** — with total return carried supplementary where dividend data exists (a labeled mix is still a mix; labeling doesn't make bases comparable). **Terminal events are typed and scored per type, never dropped**: an acquisition realizes the label at the final trading price (which converges to the deal consideration), a **bankruptcy scores to zero** (the last trade routinely overstates what equity recovers), and an ambiguous delisting (exchange removal, going-private, foreign delist) is marked **`terminal-unscorable`** — excluded from the spread but **counted and logged**, so exclusions can't silently accumulate. This matters most for the shadow ledger, whose most informative entries (the genuine losers) disproportionately delist, and silently losing *or overstating* them would bias the spread. Sector classification and benchmarks use current mappings — an accepted, *stated* approximation rather than point-in-time reconstruction.

Their purpose is to let the job **calibrate its own archetype weights and gate thresholds from realized misses** — e.g. if disruptor-archetype ideas systematically suffer `multiple-unwind`, that archetype's valuation gate tightens; and, symmetrically from the shadow side, if false-negative flags cluster on one gate or quota (the inflection gate rejecting one archetype's anchors, a diversity ceiling displacing one sector's winners), that constraint is what the calibration loosens — the picked-vs-rejected spread is the check that a tightened gate buys real selectivity rather than just fewer ideas. That **calibration loop is deliberately staged as forward work**: the labels are computed and surfaced from the first runs, but they **never auto-apply — realized outcomes *propose* weight / gate changes for the user's review**, and no proposal is even made until a statistically meaningful sample has accumulated — *statistically meaningful* is itself drafted concretely: a proposal cohort needs **≥ 30 unique issuers with matured windows**, episodes clustered by issuer, never raw episode counts — an *eligibility* bar, not proof of significance, so a proposal also reports its **effect size and an issuer-clustered uncertainty interval** for the user to judge — early runs remain **shadow / calibration** ([§The pipeline](#the-pipeline)), guarding against overfitting the gates to a handful of outcomes. Until then the lens weights stay archetype-conditioned but stable run to run ([§The research method](#the-research-method)), and the labels are recorded and audited as the job's honest scorecard — whether that scorecard gets its own display surface is a deferred decision (the same deferral Portfolio's scorecard carries — [portfolio-analysis.md §Outcome learning](portfolio-analysis.md#outcome-learning-calibration)).

## Archived opportunities

When an opportunity leaves the matrix it is **archived, not dropped** — a price-tracked record of the job's departed picks, so "what happened to the ideas you showed me" is answerable rather than silently lost. The archive holds the **most recent 100** departed opportunities (oldest evicted first).

**One way out, and it is always a failed deep re-evaluation.** An opportunity stays an opportunity until a **deep** re-evaluation judges it is no longer one — there is **no "target met" / "didn't meet target" exit**, and hitting the price prediction with upside still remaining keeps it live. The single archive trigger, `failed-reevaluation`, fires **only from a deep pass** that judges the name `invalidated` ([trade-opportunities-workflow.md §Step 7](trade-opportunities-workflow.md#step-7-continuity-check--carry-forward)) — and a deep pass reaches a live name three ways:

- **inside DTO, by the reserved rotation slice** — each run spends a small configured share of the deep budget (floored at one slot) on live opportunities in **maintenance-priority order** (warning-bearing names first — a tripped falsifier or continuation break never queues behind an uneventful stale name — then catalyst proximity, then threshold-near, then stalest), backstopped by a **max-age service level**, so every live name is re-deep-researched on a bounded cadence without depending on chance re-surfacing or user diligence;
- **inside DTO**, when the name **re-surfaces through discovery and wins leftover deep-research budget** into the Step-5 loop (the oldest-deep-researched re-surfacers first); or
- **inside ATO**, when the user **selects it for a Deep Audit**.

The deep pass is where a *qualitative* erosion the cheap read can't see (a competitive threat surfaced in research) retires a name, and equally where a deterministic exhausted-upside or continuation-failure finding is *confirmed* under fresh research rather than acted on blind. The **cheap re-derivation never archives**: when it detects exhausted upside, a continuation-failure alarm, or a tripwire, it raises the non-destructive **attention warning** ([§The opportunity](#the-opportunity)) so the user can choose a Deep Audit — turning what was an automatic exit into a prompt. The deep pass is **fail-soft**: a missing input is a gap that holds the opportunity on its last verdict, never an archival — only an affirmative failing signal under deep research archives. The cost is deliberate but bounded: the matrix is **self-flagging and slowly self-refreshing, not self-cleaning** — a name the user never re-audits and that never re-surfaces lingers (cheap-refreshed, decaying, and warning-badged) only until the rotation slice reaches it — a warning pulls it to the front of the maintenance queue, and the max-age service level bounds even the uneventful case. Staleness alone never archives a name — but it no longer accumulates unbounded either.

**Only the price is tracked afterward — nothing else.** An archived pick gets **no further analysis, no deep research, no model call**. Each run refreshes only the **price-derived components** of the since-flagged read — running return since it became an opportunity (absolute and **vs sector / market**) and maximum drawdown, **not** the leading-metric-continuation state (that needs a structured re-pull the archive deliberately skips) — the same stateless Stooq reconstruction the live matrix uses ([§Outcome learning](#outcome-learning-calibration)), so the running performance stays current at ~zero marginal cost while the rest of the record sits frozen.

**What the archive card shows is retrospective, never forward-looking.** It carries the descriptive record that *can't go stale* — thesis, archetype, leading metric, catalyst, bear case — frozen as of departure, plus `became_opportunity_at`, the departure date, the failing signal that retired it, and **conviction at exit** (a record of where it ended, not a live call). The **forward price prediction is deliberately not shown**: a frozen target beside a live price reads as a current call it no longer is. The one live element is the since-flagged return. The archive is therefore a **price-tracking record, not an outcome-graded scorecard** — the deterministic failure-mode labels (`leading-metric-rollover` / `multiple-unwind` / … — [§Outcome learning](#outcome-learning-calibration)) stay internal to the calibration loop and never surface here.

**Re-entry is a fresh start, and the archive never promotes itself.** If a later run independently re-discovers an archived name as an opportunity, it is **removed from the archive** and enters as a **new** opportunity with a new `became_opportunity_at`; **none of the archived record influences the new one** — the [anti-reflexivity / no-double-count invariant](trade-opportunities-workflow.md#step-5g-scoring--gating) the whole suite holds (the new verdict is re-derived from current fundamentals and research; the engine's stateless since-flagged price read is still used only **cap-only** at Step 5g, as for any carried name, never as an anchor or a boost). **Re-entry is matched by ticker**, deliberately — the *fresh* re-entry carries no thesis-lineage identity to match on, so the dedup key is the bare ticker, giving a clean invariant: a ticker is in exactly one state — **live** (matrix), **departed** (archive), or neither, never both; a ticker that becomes a live opportunity again vacates any archived entry for it. The one accepted edge is a **same-ticker-new-thesis** re-entry, which simply retires the prior tombstone early (the archive holds at most one slot per ticker — the most recent departure); the lost record is low-stakes price history, not a correctness break. Critically the archive is **passive**: a large archived gain never pulls a name back into analysis — that would be momentum-chasing — re-entry happens **only** through independent fresh discovery.

The archive is distinct from the two other carried stores: the **opportunity graph** holds *upstream* discovery memory (watchlist hypotheses re-checked at their metric's cadence — [§Discovery memory](#discovery-memory-the-opportunity-graph)), the **validated matrix** holds *current* picks, and the archive holds *departed* picks tracked on price alone. The calibration-only **shadow ledger** ([§Outcome learning](#outcome-learning-calibration)) sits outside this trio — it never feeds a run and never carries an idea forward; it only remembers what was turned away so the gates can be scored.

## Starting parameters (calibratable)

Trade Opportunities is not yet built; the deterministic rules and thresholds below are the **starting calibration surface** the shared engine and the discovery funnel will run on — codeable now, **shape-tested immediately against the episode library** — the pattern-lens's shipped, versioned set of dated historical metric series ([§The lenses](#the-lenses)) doubles as a regression harness: each drafted gate runs over the library's episodes as a **functional test of its shape** (a turn rule that rejects the canonical turn of its own archetype is broken), under the discipline that guards against hindsight-fitting — **constants are shaped only against the development partition, the locked holdout (ordinary failures, near misses, base-rate names) is never tuned against, and library performance is never cited as evidence the gates *work*** — then shadow-tuned against the outcome-learning scorecard ([§Outcome learning](#outcome-learning-calibration)), where realized outcomes **propose calibration changes for the user's review, never auto-apply** (early runs stay shadow / calibration). They mirror Portfolio's calibration surface ([portfolio-analysis.md §Starting parameters](portfolio-analysis.md#starting-parameters-calibratable)); the **forensic trips are the same drafted constants** that doc sets out — still pending the shared engine and the paid-FMP revision / `financial-scores` feeds — since both jobs will share one engine. The **narrative-vs-reality trips are drafted separately, one per job**: Portfolio caps conviction when multiple expansion outruns revisions by more than ~1.5×, while this job trips `hype` when multiple expansion exceeds 70% of the price move (the cap row below) — two job-specific trip rules over the same engine-computed read, not one shared constant.

| Mechanism | Starting rule (calibratable) |
|---|---|
| **Risk tier** ([§The opportunity space](#the-opportunity-space)) | High if market cap < $2B **or** realized vol > 40% **or** debt/equity > 2 **or** unprofitable **or** drawdown > 50% **or** illiquid (thin ADV / high Amihud) **or** high event-exposure; Low if market cap > $10B **and** profitable **and** debt/equity < 1 **and** vol < 25% **and** liquid; else Medium. (Inputs match Step 5h's list — profitability, market cap, liquidity, volatility, leverage, drawdown, event exposure.) |
| **Horizon** | Two typed fields, deliberately separate — **`business_runway`** (durability: how many years of high-return reinvestment remain, proxied by **low theme / TAM penetration with a still-falling unit cost**, **contracted multi-year backlog coverage**, or a typed multi-year `research_forward_assumption` — deliberately *not* the g ≈ ROIC × reinvestment formula, which estimates a growth rate, not a duration) feeding **durability and conviction**, and **`expected_thesis_realization`** (when the thesis *pays*) setting the **matrix horizon** — a durable business can still be a near-term dislocation trade. Realization, evaluated in order: **Short** if the thesis carries its own **dated, payoff-bearing catalyst < 3 months out** (the typed catalyst field with a date — *not* the mere existence of an upcoming earnings date, which every stock has); **Long** if the stated payoff mechanism **is the multi-year compounding / runway itself** (a compounder archetype or a cleared runway proxy — any archetype can earn it); **else Medium** — including an early-detection re-rate of a suppressed price, whose realization is the market's *recognition* (roughly the metric's next 1–2 reporting periods), even on a durable compounder. Never the risk inputs; archetype is a prior, not a ceiling. |
| **Hypothesis score** ([trade-opportunities-workflow.md §Step 3b](trade-opportunities-workflow.md#step-3b-model-led-hypothesis-research)) | Equal-weighted mean of {magnitude, durability, horizon-fit, metric-observability, (1 − crowding), margin-capture-clarity}, each 0–1. **Promote ≥ 0.60; watchlist ≥ 0.40.** |
| **Diversity quotas** ([trade-opportunities-workflow.md §Step 4](trade-opportunities-workflow.md#step-4-candidate-consolidation)) | Mid/small-cap **floor ≥ 40%** of slots; mega-cap **ceiling ≤ 30%**; per-feeder **≤ 50%**; per-archetype **≤ 40%**; per-sector / theme **≤ 35%**. |
| **Leading-metric "inflecting" gate** ([§The two non-negotiables](#the-two-non-negotiables)) | **Metric-family-shaped, archetype-mapped, two-phase, noise-floored** — the shape test belongs to the **metric family, declared on the metric itself** (a renewal rate is a stability-family metric whoever holds it); the archetype maps to the family its tell is expected to live in. Families: *accelerating* (segment revenue, revision velocity, net-adds — the ai-infra / disruptor / secular-compounder tells), *turned-and-holding* (a positive move after a declining stretch, or holding above a turn ≤ 4 reporting periods back, never re-demanding re-acceleration — commodity prices, ASPs; the commodity-cyclical family), *stability-under-stress / decoupling* (the metric holding within the noise band through a named, dated stressor, or operating income growing faster than revenue with the gap widening — the quality-compounder family), *threshold-crossing* (a first profit, an FCF turn), and *deterioration* (the exit shape the continuation state watches for). *Comparability:* changes are **seasonally comparable** (YoY for quarterly reported series, never QoQ on seasonal data) and trend is a **robust slope over the available points**, never the latest two deltas alone. *Continuation phase* (a continuation-mode candidate, or any re-check of an already-inflected anchor): inflected within the trailing ~4 reporting periods **and not rolled over** — steady strength passes. *Noise floor & minimum history, per family:* a qualifying move must exceed **0.5 × the σ of the series' trailing comparable changes (up to 8)**, with each family carrying its own minimum history (drafted: *accelerating* / *turned-and-holding* need **≥ 5 comparable changes** for the σ to mean anything; *stability-under-stress* needs the full stressor window; *threshold-crossing* needs the crossing plus **1 confirming observation**); below its family minimum a series is **unmeasurable** → the candidate abstains on the evidence floor unless research supplies further dated third-party observations — never a story-stock rejection. *Direction* is by **declared metric polarity** — a narrowing loss counts as improvement (the FCF-turn-while-priced-for-bankruptcy case). Always **dated / third-party-sourced**; else reject as a story stock. |
| **Forensic gate** | **Cap** conviction on Altman Z < 1.8, Piotroski ≤ 3, net income > 1.3× operating cash flow, or receivables / inventory growth > 1.5× revenue growth; **hard-exclude** on a restatement / auditor-change or a fraud flag. |
| **Archetype composite weights** ([§Archetype](#archetype--the-lens-that-decides-which-signals-matter)) | Per-archetype dominant axes, tilted ~2× over a neutral baseline: ai-infra → momentum / revision; commodity-cyclical → valuation (P/B, P/NAV); quality- / secular-compounder → quality / ROIC; disruptor → growth / leading-metric; the rest balanced. |
| **Factor normalization basis** ([trade-opportunities-workflow.md §Step 5c](trade-opportunities-workflow.md#step-5c-deterministic-analysis-archetype-weighted-engine)) | Primary: **sector-adjusted absolute bands + the company's own-history normalization** — a factor value scored against drafted per-sector bands and the name's own trailing distribution, honest without any accumulated sample. The persisted **factor-distribution store** (one **current observation per issuer** per factor, cap-band × sector bucketed) is **diagnostic-only — it never enters the score**: a selected sample of what the app happened to analyze doesn't become a market percentile at any weight, so it surfaces in the dossier / audit as context (where the candidate sits among previously analyzed names, shown once a bucket holds **≥ 20 unique issuers**, drafted) — the same held-out-until-honest pattern as the options signal. It graduates into the score **only when fed by the representative-universe snapshot** — the periodic stratified sample that is the named upgrade path (calibration-tier, unscheduled). A composite resting on thin own-history carries the **low-confidence degraded-input flag** — lowering conviction weight, never gating. |
| **Narrative "mostly multiple-expansion" cap** | When multiple expansion is **> 70%** of the price move (vs estimate revisions) — decomposed over a **trailing 12-month window**, or since `became_opportunity_at` when the idea is younger than that (the move being judged is the one the thesis rode; drafted, calibratable) — the read is `hype` → conviction cap, and an **exclude** if there is no leading-metric anchor. |
| **Forward-conviction raise (emerging-economics track)** ([§Balancing the revolutionary against the proven](#balancing-the-revolutionary-against-the-proven)) | Research may **raise** conviction by at most **one level** (low → med → high), and only on the emerging-economics track (disruptor, ai-infra) when it validates an *engine-unscored*, countable / dated / third-party leading indicator (a `research_metric` signal); bounded, cited, logged. Price action, narrative, or a metric already scored at Step 5c can **never** raise it (the no-double-count + anti-reflexivity guard); proven-economics keeps margin-of-safety. |
| **Entry asymmetry threshold** ([§The opportunity](#the-opportunity) — the gate that admits a name, and the one the upside-exhaustion read re-runs) | The engine's **base-case forward return over the twelve-month target window**, **anchored to the prevailing risk-free rate and risk-tier-scaled** — a required return of **rf + 8pts (Low) · rf + 16pts (Medium) · rf + 30pts (High)** (drafted; rf = the run-level **`DGS2`** — already loaded at Step 2, the suite's short-end anchor for the ~12-month hurdle window — a two-year yield, the already-fetched short rate rather than an exact duration match — while scenario *discounting* keeps `DGS10` for long-duration cash flows — so the bar breathes with the rate regime instead of being lax at 1% rates and unreachable at 8%). Three supplements: the **shape must not invert** — the bear-case downside may not exceed the base-case upside at admission (asymmetry, not just expected return); a **tradability-flagged name's upside is discounted for liquidity cost** before the test (an unexitable +20% is not +20%); and the **emerging-economics track keeps its own rough-double-over-horizon hurdle** ([§Balancing the revolutionary against the proven](#balancing-the-revolutionary-against-the-proven)) — the strictest binds. Every cheap re-derivation re-tests this same threshold; falling below it raises the upside-exhaustion attention warning, never an auto-archive. |
| **Upside-exhausted gate** ([§Archived opportunities](#archived-opportunities)) | The upside re-derivation flags **exhausted upside** when the re-derived **base-case forward return** over the twelve-month target window falls **below the entry asymmetry threshold** — i.e. the name would no longer clear the gate that admitted it. In the **cheap** path (DTO matrix tail / ATO Quick Audit) this raises the **attention warning**, never an archive; only a **deep** re-evaluation confirming it under fresh research archives the name. Same gate, re-run against the live price; fail-soft on missing inputs (gap holds). |
| **Forward-assumption freshness window** ([trade-opportunities-workflow.md §Step 7](trade-opportunities-workflow.md#step-7-continuity-check--carry-forward)) | A persisted `research_forward_assumption` is credited by the cheap re-derivation for **~4 weeks** after its deep pass; past it the re-derivation reverts to **structured-only** targets. Decay is **not itself an archive or amber-warning trigger** — it only stops *crediting* stale research (though it does surface the quiet, informational **Research stale** card badge — [§The opportunity](#the-opportunity) — a non-amber staleness cue). Once decay removes the prop, a name whose structured-only target **can no longer clear** the entry gate raises the **upside-exhaustion attention warning** (it does **not** archive — only a deep pass can), while one whose structured target still clears simply continues with a more conservative number. A **passing deep re-evaluation** (a rotation-slice pick or leftover-budget re-surface in DTO, or an ATO Deep Audit) writes a **fresh** assumption with a new as-of date — reaffirming, revising, or retiring the prior one — and **restarts the window**; decay runs only *between* deep passes, as a stand-in for the research that isn't happening. This same age also drives the next deep pass's **`continuity_weight`** (below). |
| **Deep-pass continuity weight** ([§The opportunity](#the-opportunity)) | When a deep re-evaluation runs on an existing opportunity, the prior record is weighted by the age of `last_deep_researched_at`, banded to mirror the freshness window: **≤ ~1 week** → *continued research* (anchor on it), **~1–4 weeks** → *blended*, **> ~4 weeks** → *fresh look* (test the prior view skeptically). It frames the model's lean on the prior record only — engine numbers are always recomputed fresh, and a name re-entering from the archive carries **no** continuity weight (a fresh start). |
| **Shadow-scorecard false-negative bar** ([§Outcome learning](#outcome-learning-calibration)) | A turned-away name (a Step-5h held-out, a dedup-collapsed peer, or a retired / unpromoted watchlist node) is flagged a **false negative** when its **market-relative** forward return exceeds **+15% at the 6-month** or **+25% at the 12-month** window; the flag is **tradability-discounted** (an untradeably illiquid "miss" is not a real miss), and flags feed gate calibration only — they never re-promote or re-surface the name (the archive's passive, anti-reflexive stance). |
| **Archive retention** ([§Archived opportunities](#archived-opportunities)) | Most recent **100** departed opportunities, oldest evicted first. |
| **Shadow matured-archive retention** ([§Outcome learning](#outcome-learning-calibration)) | Matured shadow episodes freeze into the compact archive under a drafted cap of **5,000** rows, evicted oldest-first (row-level history kept for gate-proposal re-testing — the same drafted constant as Portfolio's episode archive). |

The **configurable settings** distinct from these engine constants — the **DTO deep-research budget** (the single compute knob: how many names DTO deep-researches per run — a reserved **rotation slice** for live opportunities in maintenance-priority order first (its share and max-age service level configurable), then **new names**, then leftover on re-surfaced existing opportunities oldest-deep-researched first; ATO's depth is bounded by the user's selection, not a setting), the watchlist retention cap and carry horizon, the **shadow-ledger retention cap** ([§Outcome learning](#outcome-learning-calibration)), and the research context-management thresholds (distillation overflow, heavy-route K, per-side substantial, sub-distillation cap, seed-lineage cap) — take their **defaults at plan-time** as user-tunable knobs ([configuration.md §Local Analysis Suite Configuration](configuration.md#local-analysis-suite-configuration)), not here.

## Storage and display

Each run persists its matrix of opportunities together with an **audit record** that makes the run traceable — sources and retrieval timestamps, the discovery and screening inputs, distilled findings, the computed signals and the bear case behind each pick, each pick's conviction decomposition, the dedup-collapse decisions behind the matrix, the deterministic outcome labels on prior opportunities with each carried-forward idea's continuous since-flagged read, the shadow ledger and its scorecard, the price-target methodology, model ids and quantizations, and the prompt/schema version — with the field set specified once in [storage.md §Local Analysis Suite Storage](storage.md#local-analysis-suite-storage). The **opportunity graph** (the carried-forward discovery memory — hypotheses with value-chain traces, and watchlist nodes with their leading metric / falsifiers / status — [§Discovery memory](#discovery-memory-the-opportunity-graph)) persists **alongside the audit record as its own carried store**, reloaded at the next run's Step 2, never as an audit-record field; retention keeps the last N runs ([storage.md](storage.md)). The **archive** of departed picks ([§Archived opportunities](#archived-opportunities)) persists separately — the most recent 100, each a frozen verdict snapshot (descriptive record + conviction-at-exit + the departure date and failing signal) plus `became_opportunity_at` and `last_deep_researched_at`, with the since-flagged read recomputed statelessly each run rather than stored ([storage.md](storage.md)). The **Trade Opportunities page** renders the 3×3 matrix, each cell listing its opportunities with archetype, thesis, leading metric, catalyst, conviction, the **forward outlook / price prediction** (base-case scenario price target and bear / bull range over the twelve-month target window, shown distinct from the backward engine anchor), entry consideration, bear case, status, **its `became_opportunity_at` and `last_deep_researched_at` dates**, and — for a carried-forward idea — its **since-flagged performance** (return since it became an opportunity, absolute and vs sector / market, with the running curve and maximum drawdown); a separate **Archived opportunities** view lists the departed picks with their frozen record, departure date, and live since-flagged return — **no forward prediction** (see [interface.md](interface.md)).

The page carries **both jobs' controls**: a **Discover** button (DTO) and, gated on selection, an **Audit** button (ATO) that forks to **Quick Audit** / **Deep Audit** (a large Deep-Audit selection confirms first). Each opportunity card carries a **selection control** (with select-all / deselect-all on the matrix) and three lifecycle affordances: an amber, *actionable* **Consider Deep Audit** warning when the attention flag is set ([§The opportunity](#the-opportunity)); a green **Deep-researched today** confirmation when `last_deep_researched_at` falls on the current local-timezone day; and a quiet, *informational* **Research stale** badge (a muted, non-alarming treatment, never the amber action color) when the last deep pass is older than the ~4-week research-freshness window — the visual counterpart of the decayed `research_forward_assumption`, computed at render from `last_deep_researched_at`, no data pull.

**Matrix / List views.** The 3×3 card matrix is the page's **default and canonical** view — its risk×horizon structure is load-bearing ([§The opportunity space](#the-opportunity-space)). A **Matrix / List toggle** adds a second lens over the *same* opportunities: **List** flattens all nine cells into one ranked, sortable table (the `.ana-grid` dense-data-grid, whose sortable column headers the design system already styles), each row carrying its **risk tier and horizon** as columns so the matrix dimensions survive as metadata rather than spatial position. Sorting lives **only in the List view** (the matrix stays spatially ordered) and offers two keys, since an opportunity is **not an owned position** — there is no cost basis and no dollar gain: **forward % upside to target** (the engine's base-case twelve-month target against the current Stooq close — present on every live card) and **realized since-flagged return** (the running return since `became_opportunity_at` — [§Outcome learning](#outcome-learning-calibration) — absent on a debut pick, which therefore sorts last under this key). The **default sort is forward % upside, descending**, deliberately: it ranks by *remaining asymmetry* rather than by what has already run up, keeping faith with the suite's **cap-only, anti-momentum** stance ([§Outcome learning](#outcome-learning-calibration)) — the since-flagged key is offered but is never the default. List rows keep the same per-card **selection control and lifecycle badges** the matrix cards carry, so an ATO Audit ([§The two jobs](#the-two-jobs)) can be driven from either view. The **Archived opportunities** view — already a flat list — gains the same sort affordance (since-flagged return / maximum drawdown; default departure-date descending). The toggle and sort are **display-only** — they reorder already-computed, already-persisted fields and touch no engine, schema, or workflow — and follow frontend-craft requirements (keyboard-operable sortable headers with `aria-sort`, a stable sort, a reduced-motion reflow, and the empty-list / all-debut-pick states). The view toggle is a small **design-system extension** — a minimal tracked-caps ghost-text control (≤ 2px radius, never a pill segmented control — [interface.md](interface.md)).

The card's **price reads share one basis: the latest Stooq daily close.** The **since-flagged performance** is re-derived at render as a **keyless zero-job freshness floor** — a local recompute over the shared **per-ticker Stooq daily-bar cache**, whose lazy, fail-soft refresh rule — after-hours-only, at most once a day, both constants calibratable — is specified once in [storage.md §Local Analysis Suite Storage](storage.md#local-analysis-suite-storage), so opening the page costs no fetch once the day's bar is cached (note the cache's ET refresh basis is distinct from the user-local day the *Deep-researched today* badge uses). The displayed **% upside to target** is likewise recomputed against that same Stooq close, so one consistent current price drives every price read on the card. The live **FMP `quote`** is **not** a render dependency and not a persisted current-price field — it is a **job-time engine input** (it sets the target *level* and runs the entry gate when a cheap re-derivation or deep pass runs, logged in the audit record); the only seam is that the gate decides against the job-time FMP price while the display uses the Stooq close, which can differ by the intraday-vs-close gap at the exact threshold margin (immaterial for a thesis tool).

## Failure posture

The execution gate's **presence** preconditions are uniform across the suite — the local model daemon + roster configured ([local-models.md](local-models.md)), a connected Schwab account ([schwab-integration.md](schwab-integration.md)), and the shared FMP / FRED credentials present ([portfolio-workflow.md §Step 1](portfolio-workflow.md#step-1-job-start-and-gate)) — so missing config or a lapsed connection blocks **every** local job, Quick Audit included. One fork-aware relaxation sits at the **run-gate connectivity** layer: **ATO Quick Audit** is engine-only (no model call), so it **skips the daemon-connectivity check** and runs even when the daemon is **configured-but-down** — a lightweight refresh of an existing matrix. Its cheap re-derivation reads **no Schwab data** (the options-activity signal is held out of the grade), though the Schwab *connection* stays a presence precondition like everywhere else. **DTO and Deep Audit** take the full gate (presence + connectivity + Schwab). Web research within a run is fail-soft (thinner evidence, lower conviction), while a hard model or persistence failure fails the run ([scheduling.md](scheduling.md)). A discovery feeder that fails (a screen, a positioning scan, a theme sweep) degrades to fewer candidates rather than failing the run — this applies to **DTO**, which alone runs discovery; **ATO** has no discovery phase, operating only on the user-selected existing opportunities, and its **Quick Audit** is engine-only (no model call to fail), while a **Deep Audit** reuses the same fail-soft per-candidate research loop DTO does.
